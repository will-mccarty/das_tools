This is a pretty basic walk-through of the ncdiag.py module. It really isn't anything fancy, but it's how I interact with data. Hopefully, this serves as a use case of where power users need direct access to the data. Specifically, something between the raw data living in IODALand, but prior to a one-size-fits-all, end use GUI.

In all practical purposes, this intermediate level of interfacing with the data could also be used to generate customizable plots and tools that could be propagated upward to the GUI-level.

OK - now we want to plot the COSMIC-2 data, which has an Observation_Type between 750 & 755. Here, we can use the mask feature, which just uses a string of logic to 'select' the data.

Note: It would have been more proper to call this a select routine than a mask routine, as it returns values that match the logical string (all data points that are True )

In [6]:
msk = '(Observation_Type > 750) & (Observation_Type < 755)'
ro_file.set_mask(msk)
ro_file.use_mask(True)
sop.plot_obs_map(ro_file)

I'm lazy; typing Observation_Type is very tedious. At GMAO, we use the shorthand kx, which is also used in places within the GSI. Within the ncdiag module, I made a quick shortcut dictionary that allows shorthand:

I've also wired in the ability to do derived calculations. For example in RO space, we often look at (O-B)/B. I have a derived_var dictionary that treats calculations as variables within the file:

derived_var = {
    'amb':         {'func': ncf.amb,          'deps': ['omf','oma']},
    'fcst':        {'func': ncf.fcst,         'deps': ['omf','obs']},
...    'omfbyo':      {'func': ncf.omfbyo,       'deps': ['omf','obs']},
    'omfbyf':      {'func': ncf.omfbyf,       'deps': ['omf','obs']}
    }

So let's plot the omfbyf histogram for all used observations:
